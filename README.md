# Seasonal-Adjustment-for-Russia

Этот R-код выполняет сезонную корректировку __месячных__ временных рядов согласно российскому календарю рабочих и выходных дней, официальных праздников, а также Пасхи. Функция также позволяет зафиксировать спецификацию модели и сохранить ее для обработки новых поступающих наблюдений. 

Содержание репозитория: 
1. __sa_ru_functions.R__ - код самой функции. Его нужно скачать, но не нужно непосредственно запускать. 
2. __sa_ru_realization.R__ - код для использования функции. В нем происходит выбор настроек, запуск, диагностика, построение графиков для визуальной оценки результата. 
3. __russia_calendar.xlsx__ - файл с российским календарем. К нему обращаются оба файла с кодом.

Краткая инструкция работы с функцией: 
- Необходимо скачать все 3 файла и разместить их в одной папке.
- Откройте файл __sa_ru_realization.R__. Перед началом работы установите рабочую директорию - папку, в которую скачаны 3 файла. Это можно сделать отдельной строчкой кода или через панель "Session".
- Выберите нужную часть (см. ниже) файла __sa_ru_realization.R__, а также нужные настройки.
- Запустите нужную часть кода и сохраните результат в рабочую директорию в формате csv. 

Файл sa_ru_realization делится три части. При его запуске необходимо выбрать, что сделать: 
__Часть 1.__ Сезонно скорректировать ряд целиком. 
__Часть 2.__ Оценить параметры сезонной корректировки (и получить скорректированный ряд тоже) до определенной даты, ->
__Часть 3.__ после этой даты корректировать наблюдения с помощью уже оцененной спецификации. 
Части 2 и 3 позволяют не переоценивать исторический ряд при добавлении новых точек. Включение нового периода в оценку параметров спецификации может осуществляться, например, 1 раз в год (можно выбрать любую дату фиксации, поэтому частота пересмотра осуществляется на усмотрение пользователя). 

Настройки, доступные внутри сезонного сглаживания: 
1. __Выбор календаря__ - без учета рабочих дней, выходных и праздников / с учетом рабочих дней и выходных / с учетом рабочих дней и выходных и праздников.
2. __Включение Пасхи__ - включать Пасху и неделю перед ней / не включать Пасху и неделю перед ней.
3. __Форма преобразования рядов__ - для некоторых данных может потребоваться логарифмическое преобразование (для данных в уровнях). Для этого подойдет настройка "auto" или "log" (рекомендуется "auto"). Если необходимо сгладить данные в темпах (например, инфляцию), и ряд переведен в уровни и затем в логарифмы самостоятельно, стоит использовать вариант "none". После сглаживания темпы м./м. можно получить простым вычитанием {t+1}-{t}.
4. __Поиск выбросов__ - искать выбросы в ряде / не искать выбросы в ряде.
5. __Метод декомпозиции__ - оценка SEATS, при неудаче - автоматический переход к X11 / только SEATS / только X11
